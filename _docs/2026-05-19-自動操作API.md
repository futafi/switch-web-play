# 自動操作API (2026-05-19)

Poke-Controller的な自動操作をcurl/スクリプトから行えるよう、nxmc-serverにREST APIを追加した。コントローラー操作に加え、RTSPストリームからのスクリーンショット・音声録音も可能。

## エンドポイント一覧

| メソッド | パス | 用途 |
|---------|------|------|
| POST | `/api/input` | コントローラー操作 |
| GET | `/api/screenshot` | スクリーンショット取得 |
| GET | `/api/audio` | 音声録音 |

## POST /api/input

ボタン押下・スティック操作を送信する。`duration_ms` 指定時は自動リリース付き。

### リクエスト

```json
{
  "buttons": ["A"],
  "hat": "up",
  "lx": 128, "ly": 128,
  "rx": 128, "ry": 128,
  "duration_ms": 100
}
```

全フィールドoptional。省略時はニュートラル値（ボタンなし、hat=center、スティック=128）。

- **buttons**: ボタン名の配列。複数同時押し可
  - `A` `B` `X` `Y` `L` `R` `ZL` `ZR` `Plus` `Minus` `LClick` `RClick` `Home` `Capture`
- **hat**: 方向キー
  - `up` `upright` `right` `downright` `down` `downleft` `left` `upleft` `center`
- **lx/ly/rx/ry**: スティック値（0-255、128=中央）
- **duration_ms**: 保持時間（ミリ秒）。経過後に自動でニュートラルに戻す。0または省略時はセットだけして戻らない（set-and-hold）

### 動作

1. Reportを組み立てて `ctrl.SendReport()` で送信
2. `duration_ms > 0` の場合、指定時間スリープ後に `ctrl.Reset()` でニュートラルに戻す
3. `duration_ms = 0` の場合、セットだけして即レスポンス（連続操作やスティック保持用）

### 使用例

```bash
# Aボタンを100ms押す
curl -X POST localhost:9000/api/input -d '{"buttons":["A"],"duration_ms":100}'

# A+B同時押し
curl -X POST localhost:9000/api/input -d '{"buttons":["A","B"],"duration_ms":100}'

# 左スティックを右に倒す（手動リリース）
curl -X POST localhost:9000/api/input -d '{"lx":255}'
curl -X POST localhost:9000/api/input -d '{}'

# 十字キー上
curl -X POST localhost:9000/api/input -d '{"hat":"up","duration_ms":100}'
```

## GET /api/screenshot

RTSPストリームから1フレームをキャプチャしてJPEGで返す。

### パラメータ

| 名前 | 型 | デフォルト | 説明 |
|------|-----|-----------|------|
| width | int | (ソースサイズ) | 出力幅 |
| height | int | (ソースサイズ) | 出力高さ |

片方だけ指定した場合はアスペクト比を維持。両方省略時はソース解像度のまま。

### 使用例

```bash
# ソース解像度のまま
curl -o shot.jpg localhost:9000/api/screenshot

# 720pにリサイズ
curl -o shot.jpg 'localhost:9000/api/screenshot?width=1280&height=720'

# 幅だけ指定（高さはアスペクト比維持）
curl -o shot.jpg 'localhost:9000/api/screenshot?width=640'
```

### 実装

ffmpegでRTSP(TCP)から1フレーム取得。タイムアウト10秒。

```
ffmpeg -rtsp_transport tcp -i <rtsp-url> -frames:v 1 [-vf scale=W:H] -f image2 -c:v mjpeg -q:v 2 pipe:1
```

## GET /api/audio

RTSPストリームから音声を録音してWAVで返す。

### パラメータ

| 名前 | 型 | デフォルト | 範囲 | 説明 |
|------|-----|-----------|------|------|
| duration | int | 3 | 1-30 | 録音秒数 |

### 使用例

```bash
# デフォルト3秒
curl -o audio.wav localhost:9000/api/audio

# 10秒録音
curl -o audio.wav 'localhost:9000/api/audio?duration=10'
```

### 実装

ffmpegでRTSP(TCP)から映像なし・PCM 16bit LEのWAV出力。タイムアウトはduration+10秒。

```
ffmpeg -rtsp_transport tcp -i <rtsp-url> -vn -t <duration> -c:a pcm_s16le -f wav pipe:1
```

## サーバー設定

`--rtsp-url` フラグでRTSP URLを指定（デフォルト `rtsp://mediamtx:8554/cam`）。

```bash
# docker-compose (bridge)
./nxmc-server --rtsp-url rtsp://mediamtx:8554/cam

# host network
./nxmc-server --rtsp-url rtsp://127.0.0.1:8554/cam
```

## Dockerfile変更

ffmpegをalpineイメージに追加。

```dockerfile
FROM alpine:3.20
RUN apk add --no-cache ffmpeg
```

## 自動操作スクリプト例

```bash
#!/bin/bash
API="http://localhost:9000/api"

press() { curl -s -X POST "$API/input" -d "{\"buttons\":[\"$1\"],\"duration_ms\":${2:-100}}"; }
wait_ms() { sleep "$(echo "$1 / 1000" | bc -l)"; }

# Aボタン連打
for i in $(seq 1 10); do
  press A 50
  wait_ms 200
done

# スクリーンショットで確認
curl -s -o check.jpg "$API/screenshot?width=1280&height=720"
```
