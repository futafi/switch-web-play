# gstd導入: 解像度・ビットレート動的変更 (2026-05-19)

WebUIの解像度・ビットレート変更がダミー実装だったため、gstd (GStreamer Daemon) を導入してパイプライン再起動なしで動的変更できるようにした。

## 背景

- `gst-launch-1.0` は起動後にパイプラインのプロパティを変更できない
- 解像度・ビットレートの変更にはコンテナ再起動が必要で、映像が途切れる
- gstdはHTTP APIでパイプラインのプロパティを実行中に変更できる

## 構成

```
Browser ──POST /api/stream──> nxmc (Go)
                                │
                    PUT /pipelines/cam/elements/...
                                │
                                v
                         gstd (port 5000)
                                │
   v4l2src ! capsfilter ! jpegdec ! videoconvert ! videoscale
     ! capsfilter(name=scaler) ! x264enc(name=encoder) ! h264parse
     ! rtspclientsink ──RTSP──> mediamtx ──WebRTC──> Browser
```

## 動的変更の仕組み

- **解像度**: `videoscale ! capsfilter name=scaler` をx264encの前に挿入。gstd APIで `scaler.caps` を `video/x-raw,width=W,height=H` に差し替え
- **ビットレート**: `x264enc name=encoder` のbitrateプロパティをgstd APIで直接変更

### gstd API例

```bash
# ビットレート変更
curl -X PUT 'http://127.0.0.1:5000/pipelines/cam/elements/encoder/properties/bitrate?name=4000'

# 解像度変更
curl -X PUT 'http://127.0.0.1:5000/pipelines/cam/elements/scaler/properties/caps?name=video/x-raw,width=1280,height=720'
```

## ファイル構成

| ファイル | 内容 |
|---------|------|
| `streaming/Dockerfile.gstd` | restreamio/gstreamer上にgstd-1.xをビルド |
| `streaming/entrypoint.sh` | gstd起動 + パイプライン作成 + PLAYING |
| `nxmc/cmd/nxmc-server/main.go` | `/api/stream` でgstd HTTP APIを呼び出し |
| `docker-compose.yml` | gstreamerサービスをgstdベースに変更 |

## gstdビルド依存

`restreamio/gstreamer:latest-dev` に追加で必要なパッケージ:

- libdaemon-dev, libjansson-dev, libedit-dev, libsoup2.4-dev
- meson, ninja-build, pkg-config, build-essential

テスト・examples・systemdはmesonオプションで無効化: `-Denable-tests=disabled -Denable-examples=disabled -Denable-systemd=disabled`

## 備考

- デフォルトはDockerブリッジネットワーク（`docker-compose.yml`）。`docker-compose.host.yml` でhost modeも使用可能
- `streaming/docker-compose.yml`（スタンドアロン版）は旧gst-launch-1.0のまま未更新
