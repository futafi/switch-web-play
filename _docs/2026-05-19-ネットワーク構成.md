# ネットワーク構成 (2026-05-19)

Docker ComposeのネットワークモードにブリッジとHostの2パターンを用意した。

## ブリッジモード (デフォルト)

```bash
docker compose up -d
```

- `docker-compose.yml` を使用
- `.env` の `HOST_IP` でWebRTC ICE candidateに含めるIPを指定
- カンマ区切りで複数指定可: `HOST_IP=192.168.1.10,10.8.0.1`
- mediamtxの `MTX_WEBRTCADDITIONALHOSTS` 環境変数に渡される
- gstreamerからmediamtxへのRTSP接続はDockerサービス名 `mediamtx` で解決

### 用途

- 接続元のネットワークが限定されている場合（LAN内のみ等）
- VPN経由のアクセスがある場合はVPNのIPも `HOST_IP` に追加する

## Hostモード

```bash
docker compose -f docker-compose.host.yml up -d
```

- `docker-compose.host.yml` を使用
- 全サービスが `network_mode: host` で動作
- mediamtxがホストの全ネットワークインターフェースから自動でIPを取得（`MTX_WEBRTCIPSFROMINTERFACES=yes`）
- `HOST_IP` の設定は不要
- gstreamerからmediamtxへのRTSP接続は `127.0.0.1` を使用（`RTSP_HOST` 環境変数）

### 用途

- ネットワーク構成が複雑で全IPの列挙が難しい場合
- ポートマッピングの管理を省きたい場合

## entrypoint.shのRTSP接続先

`streaming/entrypoint.sh` は `RTSP_HOST` 環境変数でRTSP接続先を切り替える:

- ブリッジモード: `RTSP_HOST=mediamtx`（デフォルト、Dockerサービス名で解決）
- Hostモード: `RTSP_HOST=127.0.0.1`（`docker-compose.host.yml` で設定済み）
